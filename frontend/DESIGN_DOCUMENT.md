# 有道云导入功能前端设计文档

## 设计概述

### 设计目标
为知识管理平台设计一个直观、高效的有道云笔记导入功能，将有道云导入图标放置在上传文件图标的左边，突出其重要性。

### 设计理念
- **用户优先**：将最常用的导入方式放在最显眼的位置
- **视觉层次**：通过图标位置和颜色区分不同导入方式
- **一致性**：与现有设计风格保持统一
- **简洁高效**：减少用户操作步骤，提升导入效率

```
│  └─────────────────────────────────────┘│
│                                         │
│  ┌─────────────────────────────────────┐│
│  │                                     ││

│  │                                     ││
│  └─────────────────────────────────────┘│

```

### 2. 图标位置设计

**导入选项卡**：
```
┌─────────────────────────────────────────┐
│  [有道云] [文件上传] [网址导入]          │
│   ^         ^           ^               │
│   |         |           |               │
│   1         2           3               │
└─────────────────────────────────────────┘
```

**图标顺序**：
1. **有道云导入** - 最左边（优先级最高）
2. **文件上传** - 中间
3. **网址导入** - 最右边

### 3. 色彩方案

#### 主色调
- **主色**：`#3B82F6` (蓝色) - 代表信任和专业
- **辅助色**：`#10B981` (绿色) - 代表成功和导入
- **背景色**：`#F8FAFC` (浅灰) - 保持简洁

#### 状态颜色
- **激活状态**：`bg-accent text-white`
- **默认状态**：`text-text-muted hover:text-text-primary`
- **成功状态**：`text-success`
- **错误状态**：`text-error`

## 交互设计

### 1. 标签切换

- **点击标签**：切换到对应导入方式
- **视觉反馈**：激活标签显示蓝色背景
- **状态保持**：记住用户上次选择的标签

### 2. 有道云导入流程

```
1. 点击"有道云"标签
2. 显示有道云导入面板
3. 浏览目录结构
4. 选择要导入的笔记
5. 点击"导入选中笔记"
6. 显示导入进度
7. 导入完成，关闭模态框
```

### 3. 文件上传流程

```
1. 点击"文件上传"标签
2. 显示文件上传区域
3. 拖拽文件或点击选择
4. 显示上传进度
5. 上传完成，关闭模态框
```

### 4. 网址导入流程

```
1. 点击"网址导入"标签
2. 显示网址输入框
3. 输入网页地址
4. 点击"导入"按钮
5. 显示导入进度
6. 导入完成，关闭模态框
```
