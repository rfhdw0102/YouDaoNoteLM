# 系统配置动态加载功能验证清单

## 验证前准备

### 环境准备

- [ ] MySQL 数据库已启动
- [ ] Redis 已启动（可选）
- [ ] Go 环境已配置
- [ ] Node.js 环境已配置
- [ ] 项目代码已克隆

### 数据准备

- [ ] 数据库 `youdao` 已创建
- [ ] 数据库用户权限正确
- [ ] 配置文件 `config.yaml` 存在

## 验证步骤

### 第一阶段：数据库初始化

#### 1.1 运行种子脚本

```bash
cd ../YoudaoNoteLM
go run cmd/seed_sysconfig/main.go
```

**验证点**：
- [ ] 脚本运行无错误
- [ ] 输出显示配置已写入
- [ ] 显示 `asr`、`search`、`embedding` 分组

#### 1.2 检查数据库表

```sql
USE youdao;
DESCRIBE sys_config;
```

**验证点**：
- [ ] `sys_config` 表存在
- [ ] 表结构包含以下字段：
  - `id`
  - `config_group`
  - `config_key`
  - `config_value`
  - `enabled`
  - `description`

#### 1.3 检查配置数据

```sql
SELECT config_group, config_key, enabled FROM sys_config;
```

**验证点**：
- [ ] 存在 `asr` 分组的配置
- [ ] 存在 `search` 分组的配置（可能为空）
- [ ] 存在 `embedding` 分组的配置（可能为空）
- [ ] `enabled` 字段为 `1`

### 第二阶段：后端服务验证

#### 2.1 启动后端服务

```bash
cd ../YoudaoNoteLM
go run cmd/server/main.go
```

**验证点**：
- [ ] 服务启动成功
- [ ] 显示 "HTTP 服务器启动" 日志
- [ ] 端口 8080 已监听

#### 2.2 检查配置加载

查看后端日志，确认配置加载成功：

**验证点**：
- [ ] 显示 "配置加载成功" 日志
- [ ] 显示 "数据库迁移完成" 日志
- [ ] 无错误日志

#### 2.3 测试 API 接口

```bash
# 获取管理员 token
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@example.com","password":"password"}'

# 获取配置状态
curl -H "Authorization: Bearer <token>" \
  http://localhost:8080/api/v1/admin/config/status

# 获取 ASR 配置
curl -H "Authorization: Bearer <token>" \
  http://localhost:8080/api/v1/admin/config/asr
```

**验证点**：
- [ ] 登录接口返回 token
- [ ] 配置状态接口返回成功
- [ ] ASR 配置接口返回配置数据

### 第三阶段：前端服务验证

#### 3.1 启动前端服务

```bash
cd ../YouDaoNoteLM_Web
npm run dev
```

**验证点**：
- [ ] 前端服务启动成功
- [ ] 显示本地地址 `http://localhost:5173`
- [ ] 无编译错误

#### 3.2 访问后台管理页面

1. 打开浏览器访问 `http://localhost:5173/admin`
2. 登录管理员账号

**验证点**：
- [ ] 页面可正常访问
- [ ] 登录成功
- [ ] 显示后台管理界面

#### 3.3 查看系统配置

1. 点击"系统配置"标签页
2. 查看配置分组

**验证点**：
- [ ] 显示三个分组：搜索引擎、语音识别、向量化
- [ ] 每个分组有配置数量显示
- [ ] 点击分组可查看配置详情

#### 3.4 查看配置详情

1. 点击"语音识别"分组
2. 查看配置列表

**验证点**：
- [ ] 显示配置列表
- [ ] 配置名称显示正确
- [ ] 敏感信息显示为遮罩（如 `••••••••`）
- [ ] 配置状态显示正确（已启用/已禁用）

- [ ] 刷新页面后修改已保存

#### 4.2 测试配置启用/禁用

2. 确认状态改变

**验证点**：
- [ ] 开关状态改变
- [ ] 配置状态显示更新
- [ ] 刷新页面后状态已保存

#### 4.3 测试配置删除

1. 点击配置右侧的删除按钮
2. 确认删除

**验证点**：
- [ ] 删除确认对话框显示
- [ ] 删除成功
- [ ] 配置从列表中移除
- [ ] 刷新页面后配置不存在

#### 4.4 测试配置添加

1. 点击"添加配置"按钮
2. 选择服务商
3. 填写配置信息
4. 点击添加

**验证点**：
- [ ] 添加表单正常显示
- [ ] 服务商列表正确
- [ ] 配置字段正确
- [ ] 添加成功
- [ ] 配置出现在列表中

### 第五阶段：集成验证

#### 5.1 测试配置优先级

1. 创建用户配置
2. 创建系统配置
3. 测试使用哪个配置

**验证点**：
- [ ] 用户配置优先级高于系统配置
- [ ] 禁用用户配置后使用系统配置
- [ ] 无配置时使用默认配置

#### 5.2 测试配置动态更新

1. 修改系统配置
2. 不重启服务
3. 测试使用新配置

**验证点**：
- [ ] 配置修改后立即生效
- [ ] 无需重启服务
- [ ] 缓存自动失效

#### 5.3 测试配置缓存

1. 连续请求配置多次
2. 查看日志

**验证点**：
- [ ] 首次请求从数据库加载
- [ ] 后续请求从缓存加载
- [ ] 缓存命中时间 < 10ms
