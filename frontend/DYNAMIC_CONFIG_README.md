# 系统配置动态加载功能

## 概述

系统配置（搜索引擎、语音识别、向量化）现在从数据库动态加载，而不是在配置文件中写死。修改配置后，系统会自动使用最新的配置。

## 支持的服务类型

- **search**: 搜索引擎配置
- **asr**: 语音识别配置
- **embedding**: 向量化配置

## 使用方法

### 1. 初始化数据库配置

运行种子脚本将配置文件中的服务配置导入数据库：

```bash
cd YoudaoNoteLM
go run cmd/seed_sysconfig/main.go
```

### 2. 管理系统配置

访问后台管理页面：

```
/admin
```

在"系统配置"标签页中，可以：

1. **查看配置**: 选择对应的服务类型查看配置
2. **编辑配置**: 修改配置值（JSON 格式）
3. **启用/禁用配置**: 控制配置是否生效
4. **删除配置**: 删除不需要的配置
5. **添加配置**: 添加新的服务配置

### 3. 配置格式

配置值为 JSON 格式，示例：

**搜索引擎配置**:
```json
{
  "name": "serper",
  "api_url": "https://google.serper.dev/search",
  "api_key": "your-api-key"
}
```

**语音识别配置**:
```json
{
  "provider": "aliyun_nls",
  "access_key_id": "your_access_key_id",
  "access_key_secret": "your_access_key_secret",
  "app_key": "your_app_key"
}
```

**向量化配置**:
```json
{
  "provider": "openai",
  "api_url": "https://api.openai.com/v1",
  "api_key": "your-api-key",
  "model": "text-embedding-3-small"
}
```

## 配置优先级

1. **用户配置**（最高优先级）: 用户在设置页面配置的
2. **系统配置**（本功能）: 管理员在后台配置的
3. **默认配置**（最低优先级）: 代码中的默认配置

## 工作原理

1. 应用启动时，配置服务会从数据库加载系统配置
2. 当用户请求服务时，系统会按优先级查找配置：
   - 先查找用户配置
   - 如果没有，查找系统配置
   - 如果还没有，使用默认配置
3. 配置会被缓存，修改后会自动清除缓存

## 注意事项

1. 修改配置后，系统会自动使用最新配置
2. 敏感信息（如密钥）在前端显示为遮罩
3. 建议将敏感信息存储在数据库中，而不是配置文件中
4. 配置值必须是有效的 JSON 格式

## 故障排除

### 配置未生效

1. 检查数据库中是否存在对应的配置记录
2. 检查配置记录的 `enabled` 字段是否为 `true`
3. 检查配置值的 JSON 格式是否正确
4. 清除浏览器缓存或等待缓存过期

### 无法加载配置

1. 检查数据库连接是否正常
2. 检查 `sys_config` 表是否存在
3. 查看后端日志是否有错误信息
