# 系统配置动态加载功能验证指南

## 验证步骤

### 步骤 1: 初始化数据库配置

首先运行种子脚本，将配置文件中的服务配置导入数据库：

```bash
cd ../YoudaoNoteLM
go run cmd/seed_sysconfig/main.go
```

**预期输出**：
```
✅ [asr/aliyun_nls] 阿里云智能语音 ASR 服务
✅ [markitdown/default] MarkItDown 文档转换服务
✅ [storage/minio] MinIO 对象存储服务

所有系统配置已写入 sys_config 表
```

### 步骤 2: 验证数据库配置

检查数据库中是否存在配置记录：

```sql
-- 连接到 MySQL 数据库
USE youdao;

-- 查看所有系统配置
SELECT * FROM sys_config;

-- 查看特定分组的配置
SELECT * FROM sys_config WHERE config_group = 'asr';
SELECT * FROM sys_config WHERE config_group = 'search';
SELECT * FROM sys_config WHERE config_group = 'embedding';
```

**预期结果**：
- 应该能看到 `asr`、`search`、`embedding` 分组的配置记录
- 每条记录的 `enabled` 字段应该为 `true`

### 步骤 3: 启动后端服务

```bash
cd ../YoudaoNoteLM
go run cmd/server/main.go
```

**预期输出**：
```
=========================================
欢迎使用 youda-api
版本: 1.0.0
模式: debug
配置加载成功
=========================================
开始数据库迁移...
数据库迁移完成
HTTP 服务器启动 addr=:8080 mode=debug
```

### 步骤 4: 启动前端服务

```bash
cd ../YouDaoNoteLM_Web
npm run dev
```

**预期输出**：
```
  VITE v8.0.16  ready in 500 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/
```

### 步骤 5: 测试后台管理页面

1. **访问后台管理页面**：
   - 打开浏览器访问 `http://localhost:5173/admin`
   - 登录管理员账号

2. **查看系统配置**：
   - 点击"系统配置"标签页
   - 应该能看到三个分组：搜索引擎、语音识别、向量化

3. **查看配置详情**：
   - 点击"语音识别"分组
   - 应该能看到阿里云 ASR 的配置
   - 敏感信息（如密钥）应该显示为遮罩

4. **测试配置编辑**：
   - 点击配置右侧的编辑按钮
   - 修改配置值（如修改 `app_key`）
   - 点击保存
   - 刷新页面，确认修改已保存

5. **测试配置启用/禁用**：
   - 点击配置右侧的开关按钮
   - 确认配置状态已改变
   - 刷新页面，确认状态已保存

### 步骤 6: 测试配置优先级

1. **创建用户配置**：
   - 访问设置页面 `/settings`
   - 添加一个语音识别配置
   - 使用不同的 `app_key`

2. **测试优先级**：
   - 使用语音识别功能
   - 应该使用用户配置（而不是系统配置）

3. **禁用用户配置**：
   - 在设置页面禁用刚才创建的配置
   - 再次使用语音识别功能
   - 应该使用系统配置

### 步骤 7: 测试配置动态更新

1. **修改系统配置**：
   - 在后台管理页面修改语音识别的 `app_key`
   - 保存配置

2. **测试动态更新**：
   - 不重启后端服务
   - 使用语音识别功能
   - 应该使用新的 `app_key`

### 步骤 8: 测试配置缓存

1. **查看缓存日志**：
   - 在后端日志中查找缓存相关日志
   - 应该能看到缓存命中/未命中的日志

2. **测试缓存失效**：
   - 修改配置后
   - 立即使用服务
   - 应该能获取到最新配置

## 故障排除

### 问题 1: 种子脚本运行失败

**可能原因**：
- 数据库连接失败
- 表不存在
- 权限不足

**解决方案**：
1. 检查数据库连接配置
2. 确保数据库已创建
3. 检查用户权限

### 问题 2: 后台管理页面无法访问

**可能原因**：
- 未登录
- 权限不足
- 路由配置错误

**解决方案**：
1. 确保已登录管理员账号
2. 检查用户角色
3. 检查路由配置

### 问题 3: 配置修改后未生效

**可能原因**：
- 缓存未失效
- 配置格式错误
- 服务未重新加载配置

**解决方案**：
1. 清除缓存或等待缓存过期
2. 检查配置 JSON 格式
3. 重启后端服务

### 问题 4: 配置显示为遮罩

**可能原因**：
- 字段名包含 "key" 或 "secret"
- 这是正常的安全措施

**解决方案**：
- 这是预期行为，敏感信息应该显示为遮罩

## API 测试

### 获取系统配置状态

```bash
curl -H "Authorization: Bearer <token>" http://localhost:8080/api/v1/admin/config/status
```

**预期响应**：
```json
{
  "code": 0,
  "data": {
    "groups": [
      {"group": "asr", "count": 1, "enabled": 1, "description": "语音识别"},
      {"group": "search", "count": 0, "enabled": 0, "description": "搜索引擎"},
      {"group": "embedding", "count": 0, "enabled": 0, "description": "向量化"}
    ]
  }
}
```

```bash
curl -H "Authorization: Bearer <token>" http://localhost:8080/api/v1/admin/config/asr

    }
  ]
}
```

## 性能测试

### 配置加载性能

1. 添加大量配置记录（100+）
2. 测量配置加载时间
3. 确保加载时间在可接受范围内（< 100ms）

### 缓存性能

1. 连续请求配置 100 次
2. 测量平均响应时间
3. 确保缓存命中时响应时间 < 10ms

## 安全验证

### 敏感信息保护

1. 检查前端显示的敏感信息是否为遮罩
2. 检查 API 响应中是否包含敏感信息
3. 确保日志中不记录敏感信息

### 权限控制

1. 测试普通用户无法访问后台管理
2. 测试普通用户无法修改系统配置
3. 测试管理员可以正常管理配置
